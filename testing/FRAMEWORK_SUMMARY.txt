================================================================================
ARCHITECTURE OVERVIEW
================================================================================

Validation Pipeline:
  1. Email Parser → Extract headers, body, URLs, attachments
  2. Header Analyzer → Check auth, return-path, originating IP
  3. Body Analyzer → Detect urgency, CTA, credentials, obfuscation
  4. URL Analyzer → Check IP address, shortener, domain age, risk
  5. Attachment Analyzer → Detect executables, macros, MIME mismatch
  6. Risk Scorer → Aggregate findings, apply floors, generate score
  7. Results Exporter → Populate accuracy_validation.json

Test Runner:
  - Loads samples from D:\GitHub\EMLyzer\samples\
  - Runs through entire pipeline
  - Compares actual results vs expected results
  - Calculates accuracy metrics (TP/FP/TN/FN)
  - Saves to testing/accuracy_validation.json

Documentation:
  - VALIDATION_GUIDE.md: How to run and interpret
  - TEST_CASE_SPECIFICATIONS.md: What to expect
  - README.md: Quick start and overview
  - accuracy_validation.json: Test definitions and results

================================================================================
KNOWN LIMITATIONS & MITIGATIONS
================================================================================

Limitation 1: Typosquatting Detection
  - Issue: paypa1-support.com (l→1) not detected via fuzzy matching
  - Mitigation: Relying on return-path mismatch + auth failures
  - Future: Implement domain similarity scoring

Limitation 2: Brand Impersonation
  - Issue: Visual similarity (logo spoofing) not detected
  - Mitigation: Relying on domain mismatches instead
  - Future: Add logo/image analysis (requires OCR)

Limitation 3: Domain Reputation
  - Issue: Newly registered legitimate domains may be flagged
  - Mitigation: Combining WHOIS age with auth + other signals
  - Future: Add reputation service integration (AbuseIPDB, etc.)

Limitation 4: Behavioral Detection
  - Issue: No contextual knowledge ("PayPal would never ask for password")
  - Mitigation: Relying on pattern correlation
  - Future: Rule engine or advanced ML
